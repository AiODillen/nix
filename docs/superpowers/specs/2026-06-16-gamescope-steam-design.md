# Gamescope + Steam Integration Design

**Date:** 2026-06-16
**Status:** Approved

## Goal

Configure gamescope so that every game launched through Steam automatically runs inside it, with sane defaults for a 3440√ó1440 @ 175Hz ultrawide AMD setup, and a German keyboard layout inside gamescope's nested XWayland.

## Background

The gaming profile (`profiles/gaming/nixos.nix`) already sets `programs.steam.gamescopeSession.enable = true` and `programs.gamescope.enable = true`, but with no args or env configuration ‚Äî effectively untuned defaults. The `gamescopeSession` module creates a `steam-gamescope` binary that runs Steam inside gamescope. Running this binary instead of `steam` from within niri gives gamescope wrapping to every game automatically, with no per-game launch option setup required.

## Files Changed

- `profiles/gaming/nixos.nix` ‚Äî expand `gamescopeSession` args, add `gamescope.capSysNice` and `gamescope.env`
- `profiles/gaming/home.nix` ‚Äî add XDG desktop entry for `steam-gamescope`

## Architecture

### `programs.steam.gamescopeSession`

The `steam-gamescope` script generated by this module runs:
```
gamescope --steam [args] -- steam [steamArgs]
```
`--steam` / `-e` is added automatically. All display and performance flags go in `args`.

### `programs.gamescope`

The `programs.gamescope` module wraps the `gamescope` binary with `makeWrapper`, baking in default env vars via `--set-default`. Setting `XKB_DEFAULT_LAYOUT=de` here propagates it to every gamescope invocation (including via `steam-gamescope`), which gamescope passes to its nested XWayland server through libxkbcommon.

`capSysNice = true` gives gamescope the `CAP_SYS_NICE` capability so `--rt` (realtime scheduling) works without running as root.

## Gamescope Args

| Flag | Value | Reason |
|------|-------|--------|
| `-W` / `-H` | `3440` / `1440` | Output resolution matches 3440√ó1440 ultrawide panel |
| `-r` | `175` | Output refresh rate matches monitor |
| `-f` | ‚Äî | Gamescope window is fullscreen within niri |
| `--adaptive-sync` | ‚Äî | VRR / FreeSync support |
| `-F` | `fsr` | AMD FSR upscaler (best choice for AMD GPU) |
| `--sharpness` | `5` | FSR sharpness midpoint (0 = max, 20 = min) |
| `--rt` | ‚Äî | Realtime scheduling for lower input latency (requires `capSysNice`) |
| `--expose-wayland` | ‚Äî | Exposes Wayland protocol so native Wayland games work |
| `--xwayland-count` | `2` | Two XWayland servers for better client compatibility (NixOS wiki recommendation) |
| `--mangoapp` | ‚Äî | MangoHUD overlay via mangoapp (recommended integration method with gamescope) |

## German Keyboard

Gamescope creates its own XWayland server and configures its keyboard via libxkbcommon. Setting `XKB_DEFAULT_LAYOUT=de` in the environment before gamescope starts causes libxkbcommon to apply the German layout to gamescope's XWayland. This is baked into the gamescope binary wrapper via `programs.gamescope.env`.

No gamescope flag exists for XKB layout configuration (confirmed from `gamescope --help` and upstream README).

## Desktop Entry

An XDG desktop entry (`steam-gamescope`) is added to home-manager so the launcher appears in the niri app launcher alongside regular Steam. The icon reuses the `steam` icon already installed by the Steam package.

## Usage

Launch `steam-gamescope` (not `steam`) from the app launcher or terminal. Every game opened from that Steam instance runs inside gamescope. Regular `steam` remains available for a gamescope-free session.

MangoHUD overlay is shown by `mangoapp` automatically; no per-game `mangohud %command%` launch option needed.
